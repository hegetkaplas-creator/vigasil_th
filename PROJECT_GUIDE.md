# CBD Vigasil — руководство по проекту

Одностраничный тайский лендинг (advertorial) для продукта **CBD Vigasil**. Статическая сборка на **Astro**, публикация через **Cloudflare Worker**. Страница может открываться как отдельный документ и во **встроенном iframe** на родительском домене.

---

## Что внутри проекта

| Область | Где лежит | Зачем |
|--------|-----------|--------|
| Сборка страницы, порядок секций | `src/pages/index.astro` | Импорты компонентов и их порядок в `<div class="page">` |
| Оболочка HTML (`<head>`, скрипты, iframe-resizer, форма) | `src/layouts/Base.astro` | Общая разметка и глобальные подключения |
| Секции лендинга | `src/components/*.astro` | Каждый блок — отдельный файл: разметка, локальные стили, при необходимости inline-скрипты |
| Глобальные стили и токены | `src/styles/global.css` | `:root` (цвета и т.п.), общие классы, анимации |
| Медиа | `public/assets/img/`, `public/assets/video/` | Логотип, фото, документы, видео |
| Конфиг Astro | `astro.config.mjs` | Статический вывод, сжатие HTML и т.д. |
| Деплой Cloudflare | `wrangler.jsonc` | Worker раздаёт содержимое `dist/` |
| CI | `.github/workflows/deploy-worker.yml` | Установка зависимостей, `astro build`, деплой |

Подробности по форме, комментариям и ограничениям iframe см. в **`PROJECT_NOTES.md`** (операционные заметки) и **`AGENTS.md`** (соглашения для разработки и границы изменений).

## Возможности: что вы можете менять сами

### 1. Порядок и состав секций

Файл: **`src/pages/index.astro`**

- Переставьте строки вида `<Hero />`, `<Problems />` и т.д., чтобы изменить порядок блоков на странице.
- Удалите компонент из разметки и удалите соответствующий `import` вверху файла, чтобы убрать секцию.
- Чтобы добавить секцию: создайте новый `src/components/Имя.astro`, импортируйте его и вставьте `<Имя />` в нужное место.

### 2. Текст, разметка и внешний вид одного блока

Файл: **`src/components/<ИмяСекции>.astro`**

- Редактируйте HTML внутри компонента.
- Блоковые стили — в `<style>` в том же файле.
- Для согласованности с дизайном используйте CSS-переменные из **`src/styles/global.css`** вместо случайных hex-цветов для «темы» страницы.

### 3. Глобальная тема и утилиты

Файл: **`src/styles/global.css`**

- Токены в `:root`, общие классы (например glass-эффекты), keyframes, утилиты для всей страницы.

### 4. Картинки и видео

- Кладите файлы в **`public/assets/`** по смыслу (`img/`, `video/`).
- В разметке ссылайтесь путями от корня сайта, например `/assets/img/...`.
- Перед добавлением ссылки убедитесь, что файл реально лежит в `public/` — иначе на проде будет битая ссылка.

### 5. Локальная разработка и проверка сборки

```bash
npm install          # один раз (или после изменения зависимостей)
npm run dev          # dev-сервер с HMR
npm run build        # проверка прод-сборки, результат в dist/
```

### 6. Деплой

- Пуш в ветку **`main`** запускает GitHub Actions: сборка и выкладка через Wrangler (см. workflow в репозитории).
- Локальный деплой возможен через скрипты в `package.json` (если настроены Wrangler и доступы) — смотрите `package.json` и документацию Cloudflare.

## Ограничения, о которых стоит знать

- **Iframe:** страница может отображаться внутри iframe на родительском сайте. Якорные ссылки `#секция` и программный скролл могут вести себя иначе, чем на «голой» вкладке; полное поведение скролла к якорям зависит и от настроек родителя (`iframe-resizer`, `inPageLinks` и т.д.). Подробнее — в `AGENTS.md` и `PROJECT_NOTES.md`.
- **Форма заявки:** URL эндпоинта, метод и идентификаторы задаются через `data-*` на `<body>` и конфигурацию в `Base.astro`. Менять endpoint и связанные атрибуты без явной необходимости и согласования не рекомендуется.
- **Копирайт и регуляторика:** тайский текст, FDA ID, формулировки про эффект — чувствительная зона; крупные правки смысла лучше согласовывать отдельно.
